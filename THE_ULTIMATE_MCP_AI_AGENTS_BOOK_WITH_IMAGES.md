## 1.1 What is Programming? (From Absolute Basics)

### The Fundamental Concept

Imagine you want to teach a robot to make a sandwich. You can't just say "make a sandwich" because the robot doesn't understand that. You need to break it down into precise, tiny steps:

```
1. Open refrigerator door
2. Reach hand toward bread shelf
3. Grab bread bag
4. Pull bread bag out
5. Close refrigerator door
6. Open bread bag
7. Remove two slices
8. Place slices on counter
... (many more steps)
```

**This is programming.** You're giving explicit, step-by-step instructions to a computer.

### Why Computers Need Instructions

![What is Programming](images/book/ch1_what_is_programming.png)
*Figure 1.1: Programming is giving explicit, step-by-step instructions to a computer*


**Humans are intelligent:**
- You: "Make me dinner"
- Friend: *figures out what you like, checks ingredients, makes something*

**Computers are fast but not intelligent:**
- You: "Make me dinner"
- Computer: ERROR - "dinner" is undefined

**With programming:**
```python
# This is Python code - don't worry about understanding it yet
# We'll explain everything step by step

def make_dinner():
    """
    This is a FUNCTION - a reusable set of instructions.

    Breaking it down:
    - def: keyword that means "define a function"
    - make_dinner: the NAME we give this function (we chose this name)
    - (): parentheses are required (we'll learn why later)
    - """: three quotes start a documentation string
    """
    check_ingredients()  # Call another function
    prepare_food()       # Call another function
    cook_food()          # Call another function
    serve_food()         # Call another function

# Now computer knows exactly what to do!
```

### What is a Computer Program?

**A computer program is:**
1. A set of instructions
2. Written in a programming language
3. That tells a computer what to do
4. Step by step
5. In a precise, unambiguous way

**Real-world analogy:**
- **Recipe**: A program for making food
- **IKEA instructions**: A program for building furniture
- **GPS directions**: A program for navigation

### Components of a Program

Every program has:

**1. INPUT** - Data going in
```python
# Example: User types their name
name = input("What is your name? ")
# 'name' is a VARIABLE that stores the input
# We'll explain variables in detail soon
```

**2. PROCESSING** - Computer does something with the data
```python
# Example: Convert name to uppercase
uppercase_name = name.upper()
# .upper() is a METHOD that converts text to uppercase
# We'll explain methods later
```

**3. OUTPUT** - Result comes out
```python
# Example: Display the result
print(f"Hello, {uppercase_name}!")
# print() is a FUNCTION that displays text
# We'll explain functions in detail
```

**Complete example:**
```python
# INPUT
name = input("What is your name? ")

# PROCESSING
uppercase_name = name.upper()

# OUTPUT
print(f"Hello, {uppercase_name}!")
```

**If user types "alice", program outputs:**
```
Hello, ALICE!
```

## 1.2 What is Python? (Complete Understanding)

**Python** is a programming language created by Guido van Rossum in 1991.

### What is a Programming Language?

Think of it like human languages (English, Spanish, etc.):

**Human languages:**
- Let humans communicate with each other
- Have grammar rules (syntax)
- Have vocabulary (words)

**Programming languages:**
- Let humans communicate with computers
- Have syntax (grammar rules)
- Have keywords (vocabulary)

### Why Python? (Not Java, C++, JavaScript, etc.)

**Python is designed for humans:**

```python
# Python - reads almost like English
if temperature > 30:
    print("It's hot!")
    turn_on_AC()
```

Compare to other languages:

```java
// Java - more complex
if (temperature > 30) {
    System.out.println("It's hot!");
    turnOnAC();
}
```

```c
// C - even more complex
if (temperature > 30) {
    printf("It's hot!\n");
    turnOnAC();
}
```

### Python Characteristics (What Makes It Special)

#### 1. High-Level Language

**Low-level (Close to computer):**
```assembly
; Assembly language - very difficult
MOV AL, 61h
ADD AL, 01h
```

**High-level (Close to human language):**
```python
# Python - very easy
number = 97
number = number + 1
```

Python handles complex computer details automatically so you can focus on solving problems.

#### 2. Interpreted Language

**What does "interpreted" mean?**

**Compiled languages (like C):**
```
Your Code ‚Üí Compiler ‚Üí Executable File ‚Üí Run
(wait)       (wait)                     (fast)
```

**Interpreted languages (like Python):**
```
Your Code ‚Üí Python Interpreter ‚Üí Run Immediately
                                 (slightly slower but instant)
```

**Advantage:** Write code, run immediately, see results, make changes, repeat.

#### 3. Dynamically Typed

**Statically typed (Java, C):**
```java
// Must declare type
int age = 25;              // age is integer, period
age = "twenty-five";       // ERROR! Can't change type
```

**Dynamically typed (Python):**
```python
# No type declaration needed
age = 25                   # age is integer
age = "twenty-five"        # Now age is string - totally fine!
age = [25]                 # Now age is list - also fine!
```

**What this means:**
- **Variable**: Container that holds data
- **Type**: Kind of data (number, text, etc.)
- **Static typing**: Type locked in when created
- **Dynamic typing**: Type can change anytime

**Breaking it down EXTREMELY:**
```python
age = 25
```

**What happens in this one line:**

1. **`age`**
   - This is a VARIABLE NAME
   - You chose this name
   - Could have been `x`, `user_age`, `years_old`, anything
   - Convention: use descriptive names

2. **`=`**
   - This is the ASSIGNMENT OPERATOR
   - NOT mathematical equals
   - Means: "put the value on right into variable on left"
   - Read as: "age GETS the value 25"

3. **`25`**
   - This is a LITERAL VALUE
   - Specifically: an INTEGER (whole number)
   - Python automatically knows it's an integer
   - No need to declare type

4. **Overall process:**
   - Python creates a variable named `age`
   - Python recognizes 25 as an integer
   - Python puts 25 into the `age` variable
   - Now whenever you write `age`, Python knows it means 25

#### 4. Object-Oriented

**Everything in Python is an object.**

**What is an object?**
- Object = thing that has data and behaviors
- Like a real-world object

**Real-world example: A car**
- **Data (attributes):** color, speed, fuel level
- **Behaviors (methods):** start(), stop(), accelerate()

**Python example:**
```python
# String is an object
name = "Alice"

# name has data (the text "Alice")
# name has behaviors (methods we can call)

# Call the upper() method
uppercase_name = name.upper()
# Result: "ALICE"
```

**Breaking down `name.upper()`:**

1. **`name`**
   - VARIABLE containing the string "Alice"
   - "Alice" is an OBJECT (specifically, a string object)

2. **`.`**
   - DOT OPERATOR
   - Means: "access something that belongs to this object"
   - In this case: access a method

3. **`upper`**
   - METHOD NAME
   - A method is a FUNCTION that belongs to an object
   - This particular method converts text to uppercase

4. **`()`**
   - PARENTHESES
   - Means: "execute this method"
   - Empty parentheses = no additional information needed

5. **Overall:**
   - Take the object `name`
   - Call its `upper` method
   - Method returns "ALICE"
   - Store result in `uppercase_name`

We'll explore objects in extreme detail in Chapter 6.

#### 5. Extensive Standard Library

Python comes with "batteries included" - hundreds of built-in tools.

**Example - working with dates:**
```python
# Without importing anything extra, Python can't do dates
# But Python includes a datetime module

import datetime
# 'import' - KEYWORD that loads a module
# 'datetime' - MODULE NAME (collection of date/time tools)

# Now we can use datetime's features
today = datetime.date.today()
# datetime.date.today() - FUNCTION CALL
# Returns today's date

print(today)  # Output: 2024-01-15 (example)
```

**Breaking down `import datetime`:**

1. **`import`**
   - KEYWORD (special word with meaning in Python)
   - Tells Python: "I want to use code from a module"

2. **`datetime`**
   - MODULE NAME
   - A module is a file containing Python code
   - This module has tools for working with dates and times
   - Comes with Python (standard library)

3. **What happens:**
   - Python finds the datetime module
   - Python loads it into memory
   - Now you can use datetime's features
   - Use it by writing `datetime.something`

**More standard library examples:**

```python
# Math operations
import math
result = math.sqrt(16)  # Square root ‚Üí 4.0

# Random numbers
import random
dice_roll = random.randint(1, 6)  # Random number 1-6

# Working with files
import os
files = os.listdir('.')  # List files in current directory

# Making web requests
import urllib.request
response = urllib.request.urlopen('http://example.com')
```

Each of these is a MODULE in Python's standard library. There are hundreds more!

### Python's Philosophy: The Zen of Python

Type this in Python:

You'll see:
```
The Zen of Python, by Tim Peters

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
...
```

**What does this mean in practice?**

**"Beautiful is better than ugly":**
```python
# Beautiful (clear, readable)
total = sum(numbers)

# Ugly (confusing, hard to read)
t=0
for n in numbers:t=t+n
```

**"Simple is better than complex":**
```python
# Simple
if x in [1, 2, 3]:
    print("Found")

# Complex (unnecessarily complicated)
if x == 1 or x == 2 or x == 3:
    print("Found")
```

**"Readability counts":**
```python
# Readable (anyone can understand)
customer_total_purchase_amount = 0
```

## 2.1 Understanding Variables at a Deep Level

### What Exactly Is a Variable?

A variable is a **named location in computer memory** that stores a value.

**Physical Reality:**
Your computer has RAM (Random Access Memory) - billions of tiny storage locations.

```
Memory (simplified):
Address:  0x001  0x002  0x003  0x004  0x005  ...
Content:  [  0]  [ 42]  [  0]  [ 97]  [  0]  ...
```

**Without variables, you'd need to remember addresses:**
```python
# Imagine if Python worked like this (it doesn't!)
store_value_at_address_0x002(42)
print_value_at_address_0x002()  # 42
```

**With variables, you use names:**
```python
# Python does the address management for you!
age = 42
print(age)  # 42
```

### The Variable Creation Process (What Actually Happens)

When you write:
```python
age = 25
```

**Python does this behind the scenes:**

**Step 1: Create the value**
- Python creates an integer object with value 25
- This object gets a memory address (e.g., 0x1234567890)

**Step 2: Create the name**
- Python creates an entry in a "name table" (dictionary)
- Entry: `"age" ‚Üí 0x1234567890`

**Step 3: Connect them**
- The name "age" now POINTS TO the object at that address

```
‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê         ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê
‚îÇ  age     ‚îÇ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ>‚îÇ Integer Object   ‚îÇ
‚îÇ (name)   ‚îÇ         ‚îÇ Value: 25        ‚îÇ
‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò         ‚îÇ Address: 0x1234  ‚îÇ
                     ‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
```

**This is crucial to understand:**
- Variables don't "contain" values
- Variables are REFERENCES to objects
- The actual value lives in memory
- The variable is just a name that points to it

### Variables Are Labels, Not Boxes

![Variables as References](images/book/ch2_variables_references.png)
*Figure 2.1: Variables are references/labels, not containers*


**Common misconception (WRONG):**
```
‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê
‚îÇ   age    ‚îÇ
‚îÇ          ‚îÇ
‚îÇ   [25]   ‚îÇ  ‚Üê Value inside variable (WRONG!)
‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
```

**Reality (CORRECT):**
```
age ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ> [25]
 ‚Üë                 ‚Üë
Label         Object in memory
```

**Why this matters:**

```python
x = [1, 2, 3]  # x points to a list
y = x          # y points to THE SAME list

x.append(4)    # Modify the list through x

print(y)       # [1, 2, 3, 4] ‚Üê y sees the change too!
```

**What happened:**
```
x ‚îÄ‚îÄ‚îê
    ‚îú‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ> [1, 2, 3, 4]
y ‚îÄ‚îÄ‚îò             ‚Üë
              Same list
```

Both variables point to the same object!

## 2.2 Naming Variables (The Complete Guide)

### Python's Naming Rules (What's Allowed)

**Rule 1: Must start with letter or underscore**

‚úÖ **Valid:**
```python
name = "Alice"       # Starts with letter
_private = 42        # Starts with underscore
Name = "Bob"         # Capital letter okay
name2 = "Charlie"    # Can have numbers after first character
```

‚ùå **Invalid:**
```python
2name = "Alice"      # SyntaxError: cannot start with number
$price = 100         # SyntaxError: $ not allowed
@username = "bob"    # SyntaxError: @ not allowed
```

**Rule 2: Can contain letters, numbers, underscores**

‚úÖ **Valid:**
```python
user_name_123 = "Alice"
UserName = "Bob"
USERNAME = "CHARLIE"
user123name = "Dave"
_internal_var = 42
__private = 100
```

‚ùå **Invalid:**
```python
user-name = "Alice"      # SyntaxError: hyphen not allowed
user.name = "Bob"        # SyntaxError: dot not allowed (that's attribute access)
user name = "Charlie"    # SyntaxError: space not allowed
```

**Rule 3: Case-sensitive**

These are ALL different variables:
```python
name = "Alice"
Name = "Bob"
NAME = "Charlie"
nAmE = "Dave"

print(name)  # Alice
print(Name)  # Bob
print(NAME)  # Charlie
print(nAmE)  # Dave
```

**Rule 4: Cannot be a reserved keyword**

Python has 35 reserved keywords that you cannot use as variable names:

```python
# These are ALL invalid variable names:
if = 5           # SyntaxError
for = 10         # SyntaxError
while = 20       # SyntaxError
def = 30         # SyntaxError
class = 40       # SyntaxError
return = 50      # SyntaxError
```

**Complete list of reserved keywords:**
```
False      await      else       import     pass
None       break      except     in         raise
True       class      finally    is         return
and        continue   for        lambda     try
as         def        from       nonlocal   while
assert     del        global     not        with
async      elif       if         or         yield
```

**How to check if a word is a keyword:**
```python
import keyword

# Check if 'if' is a keyword
print(keyword.iskeyword('if'))      # True

# Check if 'name' is a keyword
print(keyword.iskeyword('name'))    # False

# Get list of all keywords
print(keyword.kwlist)
# ['False', 'None', 'True', 'and', 'as', ...]
```

### Python Naming Conventions (Best Practices)

**Convention 1: snake_case for variables and functions**

‚úÖ **Pythonic (good):**
```python
user_name = "Alice"
total_price = 100.50
calculate_tax = lambda x: x * 0.08
```

‚ùå **Not Pythonic (but valid):**
```python
userName = "Alice"       # camelCase (used in JavaScript)
UserName = "Bob"         # PascalCase (used for classes)
user-name = "Charlie"    # kebab-case (not valid in Python!)
```

**Convention 2: UPPER_CASE for constants**

```python
# Constants (values that shouldn't change)
MAX_CONNECTIONS = 100
PI = 3.14159
DATABASE_URL = "postgresql://localhost/mydb"
API_KEY = "abc123xyz"
```

**Note:** Python doesn't enforce constants - you CAN change them:
```python
MAX_CONNECTIONS = 100
MAX_CONNECTIONS = 200  # Allowed, but you shouldn't do this!
```

The uppercase is a signal to other developers: "Don't change this!"

**Convention 3: PascalCase for classes**

```python
class UserAccount:      # Class names
    pass

class DatabaseConnection:
    pass

class HTTPRequest:
    pass
```

We'll learn about classes in Chapter 6.

**Convention 4: Single leading underscore for "internal" use**

```python
_internal_variable = "not for public use"

class MyClass:
    def __init__(self):
        self._internal = 42       # Internal to this class
        self.public = 100         # Public

    def _helper_method(self):     # Internal method
        pass

    def public_method(self):      # Public method
        pass
```

The underscore doesn't make it private (Python has no true private), but it's a convention saying "this is internal, don't use it outside this module/class."

**Convention 5: Double leading underscore for name mangling**

```python
class MyClass:
    def __init__(self):
        self.__private = 42       # Name mangling applied

obj = MyClass()
print(obj.__private)              # AttributeError
print(obj._MyClass__private)      # 42 (actual name after mangling)
```

Python transforms `__private` to `_MyClass__private` to avoid name collisions in inheritance.

### Choosing Good Variable Names

**Bad names (unclear):**
```python
a = "John Doe"

c = "john@example.com"
d = True

def f(x):
    return x * 2
```

**Good names (clear):**
```python
full_name = "John Doe"
age = 30
email_address = "john@example.com"
is_verified = True

def double_value(number):
    return number * 2
```

**Guidelines:**

**1. Be descriptive**
```python

# Good
```

**What are floats?**

**Scientific notation:**

**Float precision issues (IMPORTANT!):**

```python
# Floats are not perfectly accurate

```

**Why does this happen?**
- Computers store numbers in binary
- Some decimal numbers can't be represented exactly in binary
- Like how 1/3 = 0.333... (infinite) in decimal
- 0.1 in decimal = infinite binary: 0.0001100110011...

**How to handle this:**

**Option 1: Use rounding for comparisons**
```python

    print("Equal enough!")
```


**When to use Decimal:**
- Financial calculations (money)
- When exact precision matters
- Scientific calculations

**When float is fine:**
- Most everyday programming
- Machine learning (approximate is okay)
- Graphics and games

**Slicing (get substring):**
```python

# start: inclusive
# end: exclusive
```

- `end`: where to stop (exclusive)
- `step`: how many to skip

**Examples:**

### Booleans (bool) - True/False Values

**What are booleans?**
- Only two possible values: `True` or `False`
- Used for conditions and decision-making
- Named after George Boole (mathematician)
- First letter MUST be capitalized

**Creating booleans:**
```python
is_valid = True          # Boolean literal True
is_error = False         # Boolean literal False

# Result of expressions

```

**Breaking down** `is_adult = age >= 18`:

1. **`is_adult`** - VARIABLE NAME (you chose this)
2. **`=`** - ASSIGNMENT OPERATOR
3. **`age`** - VARIABLE containing 25
4. **`>=`** - COMPARISON OPERATOR (greater than or equal to)
5. **`18`** - INTEGER LITERAL to compare against
6. **Overall**: Compare 25 >= 18, result is True, store in `is_adult`

**Comparison operators (produce booleans):**
```python
x = 10
y = 20

# Equal to
x == y              # False (10 is not equal to 20)

# Not equal to
x != y              # True (10 is not equal to 20)

# Greater than
x > y               # False (10 is not greater than 20)

# Less than
x < y               # True (10 is less than 20)

# Greater than or equal to
x >= 10             # True (10 is equal to 10)

# Less than or equal to
x <= 5              # False (10 is not less than or equal to 5)
```

**IMPORTANT:** `==` vs `=`
```python
# ASSIGNMENT (single =)
x = 10              # Put 10 into x

# COMPARISON (double ==)

# Common mistake:

    pass

# Correct:
if x == 10:         # Comparison - returns True or False
    pass
```

**Boolean operators (combine booleans):**

**AND operator - both must be True:**
```python
has_ticket = True
has_id = True

# Truth table for AND:
True and True       # True
True and False      # False
False and True      # False
False and False     # False
```

**OR operator - at least one must be True:**
```python
is_admin = False
is_owner = True

can_delete = is_admin or is_owner    # True (one is True)

# Truth table for OR:
True or True        # True
True or False       # True
False or True       # True
False or False      # False
```

**NOT operator - reverses the boolean:**
```python
is_logged_in = True
is_guest = not is_logged_in          # False

# Examples:

not False           # True
```

**Combining operators:**
```python
age = 25
has_license = True
has_car = False

# Complex condition

# Breaking this down:
# 1. age >= 18 evaluates to True (25 >= 18)
# 2. has_license is True
# 3. has_car is False
# 4. If rental_available is True:
#    - (has_car or rental_available) = False or True = True
#    - True and True and True = True
# 5. If rental_available is False:
#    - (has_car or rental_available) = False or False = False
#    - True and True and False = False
```

**Operator precedence (order of operations):**
```python
# NOT has highest precedence, then AND, then OR
result = True or False and False
# Evaluates as: True or (False and False)
# = True or False
# = True

# Use parentheses for clarity:
result = (True or False) and False    # False
result = True or (False and False)    # True
```

**Boolean type conversion:**

**Values that are "Falsy" (convert to False):**
```python

bool(0)             # False (zero is falsy)
bool(0.0)           # False (zero float is falsy)

bool([])            # False (empty list is falsy)
bool({})            # False (empty dict is falsy)
bool(None)          # False (None is falsy)
```

**Values that are "Truthy" (convert to True):**
```python

bool(1)             # True (non-zero number is truthy)
bool(-1)            # True (any non-zero is truthy)
bool("Hello")       # True (non-empty string is truthy)
bool([1, 2, 3])     # True (non-empty list is truthy)
bool({"key": "val"})  # True (non-empty dict is truthy)
```

**Practical example:**

### Why Type Conversion is Necessary

### Converting to Integer: int()

**Function:** `int(value)`
**Purpose:** Convert value to integer

**From string:**
```python
# String containing number
```

### Floor Division (//)

**Returns integer, rounds down:**

**Breaking down** `10 // 3`:

2. **`//`** - FLOOR DIVISION OPERATOR (divide and round down)

4. **Process**: 10 √∑ 3 = 3.333..., round down to 3
5. **Result**: 3 (INTEGER)

**Regular division vs floor division:**

### Modulus (%) - Remainder

**Returns the remainder after division:**
```python

result = 15 % 4         # 3 (15 √∑ 4 = 3 remainder 3)
result = 20 % 5         # 0 (20 √∑ 5 = 4 remainder 0)
result = 7 % 2          # 1 (7 √∑ 2 = 3 remainder 1)
```

**Breaking down** `10 % 3`:
1. **`10`** - INTEGER (dividend)
2. **`%`** - MODULUS OPERATOR (get remainder)
3. **`3`** - INTEGER (divisor)
4. **Process**: 10 √∑ 3 = 3 with remainder 1
5. **Result**: 1 (the REMAINDER)

**Visual explanation:**

```
  3 goes into 10 three times (3 √ó 3 = 9)
  Remainder: 10 - 9 = 1
```

### Operator Precedence (Order of Operations)

Just like in math, Python follows an order:

**Priority (high to low):**
1. **Parentheses** `()`
2. **Exponentiation** `**`
3. **Multiplication, Division, Floor Division, Modulus** `*`, `/`, `//`, `%`
4. **Addition, Subtraction** `+`, `-`

**Examples:**
```python
# Without parentheses:
result = 2 + 3 * 4      # 14 (not 20!)
# Explanation: 3 * 4 = 12, then 2 + 12 = 14

# With parentheses:
result = (2 + 3) * 4    # 20
# Explanation: (2 + 3) = 5, then 5 * 4 = 20

# Complex example:
result = 2 ** 3 + 10 / 2 - 4
# Step 1: 2 ** 3 = 8
# Step 2: 10 / 2 = 5.0
# Step 3: 8 + 5.0 = 13.0
# Step 4: 13.0 - 4 = 9.0
# Result: 9.0

# With parentheses for clarity:
result = (2 ** 3) + (10 / 2) - 4  # Same result, more readable
```

**Best practice:** Use parentheses when in doubt, even if not needed:
```python
# Both work, second is clearer:
result = a + b * c
result = a + (b * c)    # Easier to read
```

1. **`numbers`** - VARIABLE NAME
2. **`=`** - ASSIGNMENT OPERATOR
3. **`[`** - Opening SQUARE BRACKET (starts list)
4. **`1, 2, 3, 4, 5`** - ELEMENTS separated by commas
5. **`]`** - Closing SQUARE BRACKET (ends list)
6. **Result**: Variable `numbers` points to a list object containing 5 integers

## Appendix C: Interview Preparation

**Common Questions:**

**1. Explain the difference between list and tuple.**
Answer: Lists are mutable (can be changed), tuples are immutable (cannot be changed). Lists use [], tuples use (). Lists are for collections that change, tuples for fixed collections.

**2. What is a decorator?**
Answer: A decorator is a function that modifies another function. Uses @ syntax. Common examples: @property, @staticmethod, @classmethod.

**3. Explain GIL.**
Answer: Global Interpreter Lock. Prevents multiple Python threads from executing simultaneously. Important for threading vs multiprocessing decisions.

**4. What is MCP?**
Answer: Model Context Protocol - standardized way for AI applications to connect to data sources and tools. Created by Anthropic. Enables portable, reusable integrations.

**5. Difference between tools and resources in MCP?**
Answer: Tools perform actions (have side effects), resources provide data (read-only). Tools: send_email, create_user. Resources: read_file, query_data.
